CELIPA MODERNIZATION

From Monolith to Agentic Microservices

This index captures the full Celipa mobile surface area (screens), the HTTP requests each screen makes today, and how those calls map onto our target services architecture. Celipa v2.0 introduces an Agentic AI Orchestrator (A2A protocol) that handles intelligent bill splitting, Vision-LLM receipt extraction, and autonomous workflows. All services are built in Go with gRPC and Protocol Buffers, deployed on Google Cloud Run as microservices.

Current architecture
Monolith

Celipa today uses a single Node/Express server (celipa-server) with MongoDB, where all concerns (auth, users, receipts, items, participants, OCR orchestration, payments, notifications, dashboard) live behind the same base URL https://server.celipa.co. 

// Example (current) mobile HTTP calls
POST /user/login
POST /user/signup
PATCH /user/profile
POST /receipt/create
GET  /user/receipt/:userId
GET  /user/menuitems/:receiptId
PUT  /menu/addparticipant/:userId/:menuId/:receiptId/:quantity
PUT  /receipt/confirmStatus/:userId/:receiptId
GET  /global/credentials        // Veryfi OCR
POST (Veryfi) /partner/documents/

All of these share a single DB and Socket.io instance; scaling one concern forces scaling the whole server.

Target architecture (Celipa v2.0)
Go / gRPC / Proto

Celipa v2.0 is refactored into domain-aligned microservices built in Go with gRPC and Protocol Buffers, deployed on Google Cloud Run. Services include Auth, User, Receipt, Items, Participants, Notification, Real-time, Analytics, File Storage, and the new Celipa Agent (Orchestrator) using A2A protocol for intelligent bill splitting and Vision-LLM receipt extraction. 

// Example (target) gRPC service calls
rpc Login(LoginRequest) returns (LoginResponse)
     → Auth Service (dl/iam/v1)

rpc CreateReceipt(CreateReceiptRequest) returns (Receipt)
     → Receipt Service

rpc ProcessReceiptImage(ProcessImageRequest) returns (ReceiptData)
     → Celipa Agent (A2A orchestrator)
     └─ Vision-LLM extraction + intelligent splitting

rpc AssignItem(AssignItemRequest) returns (Item)
     → Receipt Items Service

rpc ConfirmPayment(ConfirmPaymentRequest) returns (PaymentStatus)
     → Participants Service

rpc SendNotification(SendRequest) returns (SendResponse)
     → Notification Service (dl/messaging/v1 + Twilio)

// A2A Protocol (Agent-to-Agent)
rpc SendMessage(SendMessageRequest) returns (SendMessageResponse)
     → Celipa Agent (orchestrator)

All services use gRPC with Protocol Buffers for type-safe, efficient communication. The Celipa Agent orchestrates complex workflows using A2A protocol, enabling agent-to-agent communication and autonomous task execution.

New Celipa-specific services (v2.0)
Go / gRPC / Proto

The remainder of Celipa's backend remains Celipa-specific and will be implemented as new target services in Go with gRPC and Protocol Buffers, deployed on Google Cloud Run:

  • Celipa Agent (Orchestrator) – A2A protocol-based agent that orchestrates intelligent bill splitting, Vision-LLM receipt extraction (replaces traditional OCR), proportional tax/tip distribution, shared item management, and autonomous workflows. This is the "brain" of Celipa v2.0.
  • Receipt Service – create/read/update receipts, generate receipt codes, gRPC endpoints rpc CreateReceipt, rpc GetReceipt.
  • Receipt Items Service – menu items and assignment APIs, e.g. rpc AssignItem, rpc ListItems.
  • Receipt Participants Service – participants, join codes, payment status, e.g. rpc AddParticipant, rpc ConfirmPayment.
  • Notification Service – composes the reusable email service with Twilio SMS and push notifications for Celipa-specific templates.
  • Real-time Service – WebSocket/gRPC streaming hub for receipt rooms and payment updates.
  • Analytics Service – aggregates user and receipt statistics for the mobile home dashboard and any future admin dashboards.
  • File Storage Service – GCS-backed storage for receipt images and profile pictures.

Note: Traditional OCR service is replaced by the Celipa Agent's Vision-LLM extraction capability. The agent uses multimodal AI to handle crumpled, faded, and handwritten receipts with zero-shot semantic understanding. Optionally, the existing dl/gateway/v1 service can be reused as an API gateway.

Celipa Agent (A2A Orchestrator)

The Celipa Agent is an A2A (Agent-to-Agent) protocol-based orchestrator that serves as the intelligent core of Celipa v2.0. Built in Go with gRPC and Protocol Buffers, it handles complex bill splitting workflows, Vision-LLM receipt extraction, and autonomous task execution.

Agent Architecture
A2A Protocol

The Celipa Agent implements the A2A protocol (similar to orchestrator agent), enabling standardized agent-to-agent communication, task management, and autonomous workflows. It publishes an AgentCard describing its capabilities and supports both gRPC and JSON-RPC transports. 

// AgentCard definition
agentCard := a2a.AgentCard{
    Name: "celipa-orchestrator",
    Capabilities: a2a.AgentCapabilities{
        PushNotifications: true,
        Streaming:         true,
    },
    PreferredTransport: a2a.TransportProtocolGRPC,
    Skills: []a2a.AgentSkill{
        {
            ID:          "IntelligentBillSplitting",
            Name:        "Intelligent Bill Splitting",
            Description: "Orchestrates proportional tax/tip distribution, shared items, and autonomous reconciliation",
        },
        {
            ID:          "VisionLLMExtraction",
            Name:        "Vision-LLM Receipt Extraction",
            Description: "Multimodal AI parser for receipts (crumpled, faded, handwritten)",
        },
        {
            ID:          "AutonomousWorkflows",
            Name:        "Autonomous Workflows",
            Description: "Passive ingestion, intelligent recommendations, voice-enabled splitting",
        },
    },
}
Core Capabilities
Intelligent Orchestration

1. Vision-LLM Receipt Extraction

Replaces traditional OCR (Veryfi/AWS Textract) with a multimodal AI parser that handles:

  • Crumpled, faded, and handwritten receipts
  • Zero-shot semantic understanding
  • Automatic item, price, tax, tip extraction
  • Vendor name and location detection

2. Intelligent Bill Splitting

Orchestrates complex splitting logic:

  • Proportional Tax/Tip: Distributes tax, tip, service fees based on individual item consumption
  • Shared Items: Seamless UI for splitting shared appetizers, drinks, etc.
  • Fractional Portions: Automatic cost distribution (e.g., 3 people → 1/3 each)
  • Mathematical Precision: "Everyone pays only for what they ordered"
rpc SplitBill(SplitBillRequest) returns (SplitBillResponse)
// Calculates proportional distributions
// Handles shared items, tax, tip, discounts
Agent Tools
Tool-Based Architecture

The Celipa Agent exposes capabilities as "tools" that can be called by the LLM or other services. Tools include:

  • extract_receipt_data – Vision-LLM extraction from receipt image
  • calculate_proportional_split – Distributes tax, tip, fees proportionally
  • manage_shared_items – Handles shared item splitting logic
  • suggest_saved_groups – Recommends "Saved Groups" based on location/time
  • reconcile_payments – Autonomous payment reconciliation
  • initiate_payment – Triggers real-time payment rails (Pix, UPI, FedNow)

Tools are defined in Protocol Buffers and can be called by the agent's LLM (Gemini) or directly by other services via gRPC.

Service Integration
Orchestration

The Celipa Agent orchestrates calls to other microservices: 

// Agent orchestrates the full receipt flow:
1. Receives receipt image from mobile app
2. Calls Vision-LLM extraction tool
3. Creates receipt via Receipt Service gRPC
4. Creates items via Receipt Items Service gRPC
5. Calculates proportional splits (agent logic)
6. Invites participants via Participants Service gRPC
7. Sends notifications via Notification Service gRPC
8. Publishes real-time updates via Real-time Service

// All orchestrated by the agent, not the mobile app

The mobile app sends a single request to the agent; the agent handles the entire workflow autonomously, reducing app complexity and enabling intelligent decision-making.

Agentic Vision: The Future of Celipa

Celipa v2.0 embraces Agentic AI—the transition from passive tools to active, autonomous assistants that execute complex workflows without direct human intervention. This section outlines the visionary capabilities that transform Celipa from a bill-splitting app into an intelligent financial assistant.

Pillar 1: Vision-LLM Extraction
Multimodal AI

Abandon traditional OCR in favor of a multimodal AI parser capable of handling crumpled, faded, and handwritten receipts with "zero-shot" semantic understanding.

Current (v1.0)

  • Veryfi SDK / AWS Textract
  • Requires clear, high-quality images
  • Structured OCR output
  • Manual correction needed for edge cases

Future (v2.0)

  • Gemini Vision API / Claude Vision
  • Handles crumpled, faded, handwritten
  • Semantic understanding (not just text extraction)
  • Automatic correction and validation
  • Context-aware extraction (knows restaurant menus, pricing patterns)
Pillar 2: Automated Precision
Mathematical Promise

Proportional Tax/Tip Logic

Automate the distribution of all "extra" costs (tax, tip, service fees, discounts) based on individual item consumption. "Everyone pays only for what they ordered" must be the core mathematical promise. 

// Example: 3 people, $100 bill, $10 tax, $15 tip
Person A: $40 items → $4 tax, $6 tip → $50 total
Person B: $35 items → $3.50 tax, $5.25 tip → $43.75 total
Person C: $25 items → $2.50 tax, $3.75 tip → $31.25 total
Total: $125 (exactly matches bill + tax + tip)

Shared Item Management

Provide a seamless UI for "splitting the shared appetizer." Tapping an item among three people should automatically distribute the cost into three fractional portions in the state. 

// Example: $12 appetizer shared by 3 people
Each person pays: $4.00
Agent automatically calculates:
- Base: $4.00
- Proportional tax: $0.40 (1/3 of $1.20 tax)
- Proportional tip: $0.60 (1/3 of $1.80 tip)
Total per person: $5.00
Pillar 3: The Agentic AI Frontier
Touchless Transactions

The "Holy Grail" of bill splitting in 2026 is Agentic AI—the transition from passive tools to active, autonomous assistants that execute complex workflows without direct human intervention.

1. Autonomous Bill Reconciliation

AI agents autonomously collect receipts, reconcile transactions, and validate payments—similar to BILL's "touchless" processing for small businesses.

  • Monitors Gmail/Apple Wallet for restaurant receipts
  • Automatically drafts splits based on past behavior
  • Validates transactions and reconciles payments
  • Requires minimal user intervention

2. Intelligent Recommendations

Suggesting "Saved Groups" (e.g., "The Friday Lunch Crew") based on location and time of the transaction.

  • Learns user patterns (who they split with, when, where)
  • Auto-suggests participants when creating receipts
  • Remembers payment preferences (Venmo, CashApp, etc.)
  • Context-aware suggestions (e.g., "Same group as last Friday?")

3. Voice-Enabled Splitting

Using natural language processing to allow a user to simply say, "Celipa, split the wine between Sarah and me", rather than manually dragging icons.

  • Voice commands: "Split appetizer 3 ways"
  • Natural language: "Everyone pays for their own, except the wine"
  • Conversational interface: "Who had the pasta?"
  • Multi-modal: Voice + gesture + text

4. Passive Ingestion

An AI agent that monitors the user's Gmail or Apple Wallet for restaurant receipts and automatically drafts a split.

  • Gmail integration: Scans for receipt emails
  • Apple Wallet: Detects restaurant transactions
  • Auto-creates receipts with suggested splits
  • User just confirms or adjusts
Pillar 4: Unified Payment Rails
Real-Time Settlement

The future of settlement lies in the global adoption of real-time, account-to-account payment systems. In Brazil (Pix) and India (UPI), these systems have already replaced cards for the majority of routine payments. By 2026, the US is seeing a similar shift with FedNow and RTP.

Global Payment Rails

  • Pix (Brazil): Real-time, 24/7, instant transfers
  • UPI (India): Unified Payments Interface, QR-based
  • FedNow (US): Federal Reserve's real-time payment system
  • RTP (US): Real-Time Payments network

Celipa Integration

Apps that integrate directly with these real-time rails allow for instant, cost-free transfers that happen in the background. This transforms the app from a simple ledger into a "personal financial assistant" that manages spending, savings, and credit intelligently.

  • Shift from "tracking a debt" to "initiating a settlement"
  • Instant, cost-free transfers
  • Background automation
  • Financial intelligence layer

Celipa V2 Experiences: Market Analysis & Competitive Landscape

Based on an investigation of the current market and emerging technologies in 2026, there is no single consumer app that fully integrates the entire proposed Celipa 2.0 journey—from budget-aware agentic discovery and automated internet deal-scraping to final bill settlement—into one seamless workflow.

While the market is highly fragmented, several "specialized" tools exist that provide a benchmark for specific features of the Celipa 2.0 strategy.

1. Closest "Integrated" Competitors
Market Research

A few apps have begun bridging the gap between discovery and splitting, though they lack the agentic "touchless" automation proposed for Celipa 2.0.

Tiply

Positioned as a "Smart Dining Companion," it combines location-based restaurant discovery with bill-splitting features. It uses AI to suggest fair tipping and analyzes past expenses to help users make smarter dining decisions.

Yumby

A newer entrant that focuses specifically on the dual value proposition of "Discover Restaurants" and "Dine out and split bills".

American Express Dining Companion

A high-end example that uses generative AI to help cardholders discover restaurants through conversational queries, integrating spend history and the Resy API.

2. Tools with Agentic "Discovery & Budget" Features
Agent Technology

Several platforms demonstrate that the technical "agent" layer for Celipa 2.0 is already achievable.

DesiYatra (Multi-Agent System)

This project utilizes a "Scout Agent" to scan multiple internet sources (Google Maps, Search) and a "Bargainer Agent" to autonomously make calls and negotiate prices, specifically including group discount negotiation.

AgentX & Tars AI

These platforms provide "AI Restaurant Finder Agents" that process natural language queries like "What's the cheapest dish?" or "Find a spot for 10 people under $30/person" by evaluating menu data and location proximity.

Tenere Team

Employs automated systems that constantly "scan the internet" for the latest restaurant deals and discounts, which could serve as a backend model for Celipa's deal discovery.

3. Gap Analysis: Where Celipa 2.0 Can Win
Competitive Advantage

The investigation reveals a significant "User Experience Gap" that current tools have failed to solve:

The Experience Orchestrator Vision
Strategic Direction

The vision for Celipa 2.0 as an "Experience Orchestrator" aligns perfectly with the 2026 market shift toward agentic AI, where digital assistants move from providing lists to executing multi-step actions on behalf of the group. By integrating trip planning and privacy-first media management, Celipa 2.0 fills the "Experience Gap" where groups currently have to juggle fragmented tools for discovery, voting, budgeting, and photo sharing.

1. The Intelligent Trip Planner: "Goal-Oriented Orchestration"
Multi-Agent System

Instead of a standard search engine, Celipa 2.0 utilizes a Multi-Agent System to handle the "herding cats" problem of group travel.

The Scout Agent

Uses Marketplace APIs (e.g., Viator, GetYourGuide, Bridgify) to pull real-time inventory for tours and activities. It semantically parses menus and activity descriptions to ensure they fit group dietary and physical constraints.

The Bargainer Agent

Inspired by systems like DesiYatra, this agent autonomously contacts local vendors or hits discount APIs to negotiate "Volume/Group Discounts" (e.g., 10% off for parties of 8+).

Simulated Group Liability

Before any booking, the agent calculates a "Preparation to Pay" range for each individual using the proportional formula: 

Expected Share = (Base Activity Cost × (1 - Negotiated Discount)) + Est. Add-ons

This eliminates the "Unscrupulous Diner's Dilemma" (overspending by 37%) by setting clear expectations early.

2. New Group-Based Activity Focus Areas
Market Research 2026

Research into 2026 travel trends reveals high-growth sectors where a group-centric app can establish a competitive moat:

Social Wellness & Shared Rituals

There is a surge in "Social Wellness"—groups swapping nightlife for shared rituals like group saunas, sunrise runs, and meditation retreats. Celipa can integrate booking for wellness centers that offer group-sized facilities.

"Pop Culting" & Fandom Trips

Big cultural moments are driving travel (e.g., Taylor Swift tours or filming location tours). Celipa could focus on Event-Linked Itineraries, where the agent monitors ticket releases (World Cup 2026, Coachella) and automatically proposes hotel-flight-transport bundles for the group.

"Hyper Playgrounds" (Immersive Entertainment)

43% of travelers are drawn to "high-concept" experiences like escape rooms, immersive 4D dining, and VR arenas. Celipa can partner with ticketing APIs that handle complex capacity controls and time-slot management for these venues.

Group Transport Logistics

Groups frequently struggle with "who paid for the Uber?" Celipa's agent can monitor rideshare APIs and automatically split the cost of multiple 4-person cars or 12-person shuttles required to move the whole group.

3. The Collaborative File Vault: Privacy-Centric Vision
Privacy-First Architecture

Current group photo sharing is often a "dump" of images where privacy is ignored. Celipa 2.0 can differentiate using Selective Privacy AI.

Selfie-Based Discovery

Similar to TurtlePic or Kamero, guests upload a "registration selfie." The Vision Media Service then scans the trip gallery and automatically generates a personalized feed for each person.

Privacy-First Permissions

  • "In-the-Picture" Access: By default, users only see photos they are physically in.
  • Anonymous Face Scanning: To comply with 2026 privacy laws, the AI can perform "Anonymous Face Matching" where facial data is never linked to legal identities, only to a "Trip Token".
  • Automatic Meta-Tagging: The vault can utilize vision models to tag photos by location, activity (e.g., "Dining," "Hiking"), and sentiment to create an automated Group Trip Journal.

End-to-end flows (monolith → microservices)

Receipt creation flow
Scan → OCR → Create 
HomeScreen
  └─ tap "Scan receipt"
       ▼
ReceiptPictureScreen
  └─ POST image to Celipa Agent (gRPC)
       ▼
Celipa Agent (A2A Orchestrator)
  ├─ Vision-LLM extraction (multimodal AI)
  ├─ Creates receipt (Receipt Service gRPC)
  ├─ Creates items (Items Service gRPC)
  ├─ Calculates proportional splits (agent logic)
  ├─ Invites participants (Participants Service gRPC)
  └─ Returns structured data
       ▼
UserReceiptScreen
  └─ Displays extracted data (editable)
       ▼
VirtualReceiptScreen
  ├─ WS   Real-time Service (gRPC streaming)
  ├─ Agent orchestrates collaborative splitting
  └─ Proportional tax/tip calculated automatically

In Celipa v2.0, the Celipa Agent orchestrates the entire flow. The mobile app sends a single request with the receipt image; the agent handles Vision-LLM extraction, service calls, and intelligent splitting autonomously. The app flow is simpler, but the backend is more intelligent.

Join & pay flow
Join → Select → Pay 
HomeScreen
  └─ tap "Join receipt" (or scan QR code)
       ▼
JoinReceiptScreen
  ├─ gRPC: GetReceiptByCode (Receipt Service)
  └─ navigate(VirtualReceiptScreen, { receiptId })
       ▼
VirtualReceiptScreen
  ├─ Agent suggests items based on user history
  ├─ Voice: "Split appetizer 3 ways" → Agent executes
  ├─ gRPC: AssignItem (Items Service)
  ├─ Agent calculates proportional tax/tip automatically
  └─ tap "Confirm selection"
       ▼
AcceptPaymentModal
  ├─ Agent shows final amount (with proportional splits)
  ├─ gRPC: ConfirmPayment (Participants Service)
  └─ Agent initiates real-time payment (Pix/UPI/FedNow)
       ▼
ConfirmationScreen
  └─ Payment completed in seconds (not days)

With Celipa v2.0, the Celipa Agent orchestrates the flow, provides intelligent suggestions, handles voice commands, calculates proportional splits automatically, and initiates real-time payments. The user experience is seamless; the agent does the heavy lifting.

Real-time & events

Socket.io channels
Real-time Service

Celipa already uses Socket.io heavily in VirtualReceiptScreen and PaymentUpdatesScreen. The microservices migration keeps the same client code, but points the connection at a dedicated Real-time service rather than the monolith. 

// Current
const SocketEndpoint = SERVER_URL + "/";
const socket = io(SocketEndpoint, { transports: ["websocket"] });

socket.on(receiptId, info => {
  // receipt, menu, participants, tip/tax
});

socket.on(`${receiptId}pu`, info => {
  // payment updates
});
// Target
const REALTIME_URL = "wss://realtime.celipa.co";
const socket = io(REALTIME_URL, { transports: ["websocket"] });

// Event payloads emitted by Real-time Service in response
// to domain events on the message bus:
//
// - receipt.updated
// - item.assigned
// - participant.added
// - tip.updated
// - payment.confirmed

This mirrors the target services pattern where each domain emits events into a shared bus and presentation/frontends subscribe via a gateway and real-time layer.
